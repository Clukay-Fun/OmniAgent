# Deploy (独立部署架构层)

`deploy/` 目录专门用于存放 OmniAgent 系统在生产环境或本地沙盒中的**容器化编排**与**自动部署配置**。

遵循物理隔离原则，所有与环境搭建、容器启停相关的基建代码都汇聚于此，从而确保主业务代码 (`src/`) 保持绝对纯净且不受外部运维框架耦合污染。

## 目录结构解析

### 1. `docker/` (容器化编排模块)
本目录不仅是将系统进行 Docker 化的配置集大成者，也是我们向异构服务器平滑迁移的底气。
- **`compose.yml`**: **生产环境主编排文件**。定义了 Agent 核心服务、网络隔离以及依赖组件的启动拓扑结构，是多容器服务一键启动的标准入口。
- **`compose.dev.yml`**: **开发/调试覆盖配置**。当开发者需要在本地通过容器运行 Agent 但又希望保留热更新（挂载源码）能力时，与主 Compose 结合使用。

---

## 运维哲学

考虑到核心业务场景专注于轻量化协作，本项目剥离了如 Prometheus + Grafana 这样重型的本地监控容器体系。日常性能与错误排雷，可直接使用飞书开放平台提供的免配置数据底座（“分析与诊断”模块），或者通过无侵入式的外部 APM 工具（如观测云/ARMS/LangSmith）来实现大模型链路追踪与消耗诊断。

所有容器组件按需使用。在平时本地敏捷开发中你几乎不用进入 `deploy/` 目录；而一旦决定登上稳定运行的远程公有云，本目录下的 `compose` 配置就是系统平滑启航、无痛分发的最强护航舰。
