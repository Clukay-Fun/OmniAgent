# Skill: create

## 描述
负责创建新记录（如案件、任务、提醒等）并返回创建结果。
在执行前需要提取并校验最小必需字段。

## 触发条件
- 用户表达“新增/创建/录入/登记”意图
- 输入包含可落地的实体信息（如案号、当事人、时间等）
- 当前请求不是更新已有记录，也不是仅查询

## 参数
- `query`: 用户创建请求原文
- `user_id`: 用于审计、会话与权限边界
- `extra`: 可选上下文（如目标表、预填字段、回调意图）

## 约束
- 必填字段不完整时，先澄清再执行
- 写链路必须通过 locator triplet 校验：`app_token` + `table_id` 必填（`record_id` 在 create 可选）
- 严格区分“创建”与“更新”，避免误改存量记录
- 写入前进行基础数据规范化（时间、枚举、空值）
- 返回结果需包含是否成功、关键标识（如记录 ID）与失败原因

## 示例对话
用户：帮我新增一个案件，案号 2026-沪01民初100 号，当事人王某，开庭日下周三上午十点
助手：已创建案件记录（ID: rec_xxx），案号 2026-沪01民初100 号，开庭时间 2026-03-04 10:00。

用户：再建一条，只有案号
助手：可以，创建前还需要补充当事人和开庭时间。
